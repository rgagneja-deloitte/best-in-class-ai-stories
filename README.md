# Best in Class Stories

Static website for interactive Deloitte client story pages.

## Current pages

- `index.html` - landing page / story hub
- `nestle.html` - Nestle story page
- `paypal.html` - PayPal story page

## Supporting files

- `landing.css` - styles for the landing page
- `styles.css` - styles for the Nestle page
- `script.js` - interactions for the Nestle page
- `paypal.css` - styles for the PayPal page
- `paypal.js` - interactions for the PayPal page

## Publish to GitHub Pages

1. Create a new GitHub repository.
2. Upload these files to the root of the repo:
   - `index.html`
   - `landing.css`
   - `nestle.html`
   - `styles.css`
   - `script.js`
   - `paypal.html`
   - `paypal.css`
   - `paypal.js`
   - `README.md`
3. You can leave out `Customer AI&A Best-In-Class Stories_vPublish.pptx` unless you want the source deck in the repo.
4. In GitHub, open `Settings` -> `Pages`.
5. Under `Build and deployment`, choose:
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`
   - `Folder`: `/ (root)`
6. Save the settings and wait for GitHub Pages to publish.
7. GitHub will give you a public site URL.

## Updating the site later

1. Edit the files in this workspace.
2. Upload the changed files to the same GitHub repo, or commit/push them if you later connect this folder to Git.
3. GitHub Pages will republish automatically.

## Adding another story

1. Create a new HTML page for the story.
2. Add any page-specific CSS or JS files if needed.
3. Add a new card to `index.html`.
4. Move the story card from the placeholder list to the live list when the page is ready.

## Recommended next step

Create a new GitHub repo named something like `best-in-class-stories`, upload the website files above, and turn on GitHub Pages.
